# context/ — Annie's knowledge layer

Both skills load this folder on every run. It splits into two kinds of files:

**Ships with the repo (generic machinery):**
| File | Purpose |
|---|---|
| `intake.md` | Shared intake: required inputs, input discipline, anchoring rule |
| `section-a-guardrails.md` | Precedence rules, candidate truth law, verbosity limits |
| `how-to-answer-questions.md` | Answer construction rulebook |

**You create (personal, gitignored, built from the `.example.md` templates):**
| File | Purpose |
|---|---|
| `section-b-voice.md` | Your tone of voice. **Mandatory: Annie halts without it** |
| `resume-talent-leadership.md` | Resume variant 1 (rename to fit your search) |
| `resume-recruiting-ops.md` | Resume variant 2 |
| `writing-examples.md` | 2-4 real answers that calibrate your register |
| `linkedin-profile.pdf` | Your LinkedIn export (Profile → More → Save to PDF) |

Fill in each `.example.md`, save a copy without the `.example`, and Annie is ready.
The `.gitignore` guarantees your personal versions never reach the repo.
